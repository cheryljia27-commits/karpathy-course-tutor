# Product thesis

## Problem

AI tutors are increasingly good at explanation, but most still wait for the
learner to start. In self-directed study, the failure often occurs one step
earlier:

```text
the learner must decide whether to restart and where to restart
```

The lecture, code, and notes survive an interruption. The active plan often
does not.

## Core claim

```text
initiation is re-entry point selection
```

The tutor's distinctive value is not explaining everything. It is reducing
the decision cost of returning to the right learning loop.

That requires a small amount of explicit state:

- current source and focus;
- last unresolved question;
- present stuck point;
- next inspectable artifact;
- a short timebox;
- what not to reopen today.

## Why this source pack

Karpathy's public teaching material is unusually compatible with an
artifact-oriented tutor. Across courses, code, and writing, the recurring
learning mechanics are:

```text
reduce scope
→ build or inspect a tiny complete object
→ observe a failure
→ verify one step
→ name the bottleneck
→ file the learning
```

This repository applies those mechanics without imitating a person.

## Success

A re-entry intervention succeeds when:

- the learner knows exactly where to restart;
- the next action takes roughly 5–15 minutes;
- the action creates or changes a durable artifact;
- an observable check distinguishes progress from a plausible-sounding answer;
- the original course remains the source of truth.

## Non-goals

The tutor is not:

- a persona simulation;
- a replacement for lectures, code, or deep work;
- a generic reminder bot;
- a fake friend using memory to perform intimacy;
- a long-form classroom inside a messaging thread;
- a transcript archive.

The message is the doorway. The source and artifact are the classroom.
